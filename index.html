Modelo de Calidad del Producto de Software

Marco normativo internacional para evaluar y garantizar la calidad del software, definiendo características y subcaracterísticas medibles que orientan el desarrollo y la evaluación de productos de software de manera sistemática.

📘 Introducción y Contexto

ISO/IEC 25010

¿Qué es ISO/IEC 25010?

ISO/IEC 25010 es parte de la familia de normas SQuaRE (Systems and software Quality Requirements and Evaluation). Define el modelo de calidad del producto de software, estableciendo un conjunto estructurado de características y subcaracterísticas que sirven como base para especificar, medir y evaluar la calidad de cualquier sistema o producto de software, ya sea durante su desarrollo, adquisición o uso.

🎯 Objetivos del Estándar

La norma ISO/IEC 25010 tiene como objetivo principal proporcionar un modelo de referencia para la calidad del software que pueda ser utilizado por desarrolladores, adquirentes y evaluadores. Permite definir requisitos de calidad de forma precisa, establecer criterios de aceptación medibles y comparar productos de software de manera objetiva y sistemática. Su aplicación favorece la mejora continua y la comunicación entre todas las partes interesadas.

🧩 Las 8 Características de Calidad

ISO/IEC 25010
✅ 1. Adecuación Funcional

Capacidad del software para proporcionar funciones que satisfacen las necesidades declaradas e implícitas bajo condiciones específicas.

  • Completitud funcional
  • Corrección funcional
  • Pertinencia funcional
⚡ 2. Eficiencia de Desempeño

Desempeño relativo a la cantidad de recursos utilizados bajo condiciones determinadas.

  • Comportamiento temporal
  • Utilización de recursos
  • Capacidad
🔗 3. Compatibilidad

Capacidad de intercambiar información y/o realizar funciones requeridas mientras se comparte el mismo entorno.

  • Coexistencia
  • Interoperabilidad
🖥️ 4. Usabilidad

Capacidad del producto para ser usado por usuarios específicos para lograr objetivos con efectividad, eficiencia y satisfacción.

  • Reconocibilidad de idoneidad
  • Capacidad de aprendizaje
  • Operabilidad
  • Protección de errores del usuario
  • Estética de la interfaz
  • Accesibilidad
🛡️ 5. Fiabilidad

Capacidad de realizar funciones especificadas bajo condiciones determinadas durante un período de tiempo establecido.

  • Madurez
  • Disponibilidad
  • Tolerancia a fallos
  • Capacidad de recuperación
🔒 6. Seguridad

Capacidad de proteger información y datos contra accesos no autorizados, garantizando los niveles de acceso apropiados.

  • Confidencialidad
  • Integridad
  • No repudio
  • Responsabilidad
  • Autenticidad
🔧 7. Mantenibilidad

Capacidad del software para ser modificado de manera efectiva y eficiente por los mantenedores designados.

  • Modularidad
  • Reusabilidad
  • Analizabilidad
  • Capacidad de modificación
  • Capacidad de prueba
🚀 8. Portabilidad

Facilidad con la que el software puede ser transferido de un entorno hardware o software a otro.

  • Adaptabilidad
  • Capacidad de instalación
  • Capacidad de sustitución

💡 Nota sobre la versión 2023

La versión ISO/IEC 25010:2023 introduce refinamientos en las definiciones de seguridad y usabilidad, y alinea el modelo con las nuevas realidades tecnológicas como sistemas basados en IA, servicios en la nube y arquitecturas de microservicios. Se recomienda revisar periódicamente los cambios publicados por ISO/IEC JTC 1/SC 7 para mantener la alineación con la versión vigente del estándar.
